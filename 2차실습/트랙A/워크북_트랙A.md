# 🅰️ Claude Code 2차 실습 워크북 · 트랙 A
## Notion 자동화 — 주간보고를 한 줄로

> **오늘의 목표:** 내 반복업무(주간보고)를 `claude` **한 줄(또는 예약)** 로 만들어 Notion에 올리고, 동료 앞에서 시연합니다.
> **대상:** 1차에서 Claude Code 설치 + Notion 연결을 마친 분 · **소요:** 4~5시간
> 모든 STEP은 **① 복붙 블록 → ② 이렇게 나오면 성공 → ③ ✋체크포인트 → 🆘안 될 때** 순서입니다.
> 이 워크북은 **트랙 A 전용**입니다. (대시보드를 만들 분은 트랙 B 워크북으로)

---

## ✅ 시작 전 30초 점검

아래 3개가 모두 ✅면 바로 시작하세요. 하나라도 ❌면 손을 들어주세요 (조교 호출).

|---|---|---|
| Claude Code 실행되나 | 터미널에 `claude` 입력 | 대화창이 뜬다 |

| 실습 폴더 받았나 | 바탕화면 `2차실습/트랙A` 폴더 | `이번주_메모.md`가 보인다 |

> 📸 *[스크린샷 자리: Notion 프로젝트 목록이 나온 대화창]*

> 🆘 Notion 목록이 안 나오면 → `claude "Notion 다시 연결해줘"` 로 토큰 재발급 후 다시 시도.

# STEP 0 · 내 자동화 후보 찾기  ⏱ 10분

매번 다시 시키는 일 = 자동화 후보입니다. 1차 때 쌓인 내 대화 기록에서 직접 찾아봅니다.

**📋 복붙 블록 0-1** — Claude Code에 그대로 붙여넣기
```
내 ~/.claude/projects 폴더의 최근 대화 기록을 분석해서,
내가 비슷한 요청을 반복한 작업 Top 5를 찾아줘.
각각 ⓐ무슨 일을 ⓑ몇 번쯤 시켰는지 ⓒ자동화하면 절약될 시간을 표로 정리하고,
그중 '보고·정리·등록'처럼 Notion로 옮기기 좋은 일 1개를 추천해줘.
```

**② 이렇게 나오면 성공**
- "반복한 작업 Top 5" 표가 나온다. (기록이 적으면 적게 나와도 OK)

> 📸 *[스크린샷 자리: Top 5 표 결과]*

**🆘 안 될 때**
- 기록이 거의 없다 → 괜찮아요. STEP 1 설문이 후보를 대신 잡아줍니다.

# STEP 2 · 공통 레일 만들기  ⏱ 45분  ★게이트 A

> 모든 자동화는 같은 골격입니다: **① 규칙 적기(CLAUDE.md) → ② 한 줄 실행 → ③ 예약(STEP 3)**

### A-1. 규칙 만들기
`2차실습/트랙A` 폴더를 Claude Code로 연 뒤, `CLAUDE.md` 파일을 만들고 붙여넣습니다.

**📋 복붙 블록 A-1**
```markdown
# 주간보고 자동화 규칙
## WHAT: 이번 주 내 업무를 정리한 주간보고를 만든다
## 입력: ./이번주_메모.md
## 양식: 1) 금주 한 일  2) 차주 계획  3) 이슈 — 각 3줄, 존댓말
## 끝나면: Notion '주간보고' 데이터베이스에 항목으로 등록
## 검증: 끝나면 등록한 내용 3줄로 요약해서 보여줘
```

### A-2. 한 줄로 실행
**📋 복붙 블록 A-2**
```
claude "이번 주 주간보고 만들어서 Notion에 올려줘"
```

**② 이렇게 나오면 성공**
- 메모를 읽고 → 양식대로 정리 → Notion에 태스크가 등록되고 → 링크/요약이 회신된다.

> 📸 *[스크린샷 자리: Notion에 등록된 주간보고 태스크]*

### ✋ ★게이트 A — 전원 통과 지점
- [ ] **"한 줄 실행"으로 Notion에 주간보고가 한 번 올라갔다**

> 🎉 이 순간 **자동화 1개(Lv2) 완성** = 졸업 조건 충족. 안 되면 손 들기! 🙋

**🆘 안 될 때**
| 증상 | 해결 |
|---|---|
| 파일을 못 찾는다 | 파일 이름이 `이번주_메모.md` 와 같은지 / 트랙A 폴더에서 열었는지 확인 |
| Notion 등록 실패 | `claude "Notion 연결 상태 확인해줘"` → 토큰 만료면 재연결 |
| 엉뚱한 프로젝트에 올라감 | CLAUDE.md `## 끝나면` 에 프로젝트 이름을 정확히 적기 |
| 결과가 이상하다 | CLAUDE.md `## 양식` 을 더 구체적으로 (줄 수·말투·항목) |

# STEP 4 · 내 것으로 갈아끼우기  ⏱ 50분  ★게이트 C

> STEP 2에서 만든 골격에서 **입력·양식·목적지만 내 기획서(STEP 1) 값으로** 바꾸면 됩니다.

### 4-1. CLAUDE.md 3칸 바꾸기

| 바꿀 칸 | 공통 예제 | → 내 것 (기획서에서) |
|---|---|---|
| `## 입력` | 이번주_메모.md | → 내 실제 메모/기록 파일명 |
| `## 양식` | 주간보고 3항목 | → 내가 원하는 보고 양식 |
| `## 끝나면` | Notion '주간보고' | → 내가 올릴 Notion 프로젝트 |

### 4-2. 다시 한 줄 실행해서 확인
**📋 복붙 블록 4-1**
```
claude "내 주간보고 만들어서 Notion에 올려줘"   (※ 내 작업에 맞는 문장으로)
```

> 📸 *[스크린샷 자리: 내 데이터로 Notion에 올라간 결과물]*

### ✋ ★게이트 C — 내 자동화가 한 번 돌았다
- [ ] 내 메모/내 Notion 프로젝트로 결과물이 한 번 올라갔다 → **오늘의 졸업 작품!**

### 🧰 막혔다면 → 기본 키트 (둘 중 하나만 돌려도 게이트 C 통과)

**📋 기본 키트 ① · 회의록 → 할 일 등록**
```
claude "이 회의록에서 내 할 일만 뽑아 Notion에 태스크로 등록해줘"
```
**📋 기본 키트 ② · Notion 내 할 일 정리**
```
claude "Notion에서 내 이번 주 할 일 모아서 우선순위로 정리해줘"
```

> 빨리 끝났다면? → 두 번째 자동화에 도전하거나, 옆자리 멘토가 되어주세요. 🙌

# STEP 5 · 견고화 (안 깨지게)  ⏱ 35분

### 5-1. "왜 안 돼!"를 줄이는 3원칙
1. **"이거" 금지** — 대상을 1개 명시. ("이거 또 올려줘" ❌ → "이번주_메모.md로 주간보고 올려줘" ✅)
2. **한 번에 한 의도** — 지시+질문+변심을 한 줄에 섞지 않기.
3. **항상 검증** — CLAUDE.md에 `## 검증: 끝나면 등록 내용 3줄로 요약해서 보여줘`를 넣기.

### 5-2. 안전장치 체크리스트
- [ ] 민감정보(고객/개인정보/계약 단가 등)는 메모에서 뺐거나 가렸다
- [ ] 등록 후 검증 한 줄을 CLAUDE.md에 넣었다
- [ ] 예약 자동화는 **첫 2~3회 Notion 등록 결과를 눈으로 확인**하기로 했다

### 5-3. 자동화가 멈췄을 때 — 자가진단 3줄
1. **입력이 그대로 있나?** (`이번주_메모.md` 이름·위치 변경 여부)
2. **Notion 연결이 살아있나?** (토큰 만료 / 로그인 풀림)
3. **그냥 물어보기** → `claude "이 자동화 왜 안 됐는지 봐줘"`
